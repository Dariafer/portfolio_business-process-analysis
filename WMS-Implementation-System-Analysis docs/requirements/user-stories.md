Пользовательская история
US-01
Как планировщик склада
Я хочу создать запрос на перемещение запасов в ERP-системе
Чтобы операторы склада получали официальное задание.

US-02
Как оператор склада
Я хочу видеть назначенные задачи по перемещению на своем мобильном устройстве (TSD)
Чтобы я мог выполнять их без бумажных документов.

US-03
Как оператор склада
Я хочу сканировать штрих-коды товаров перед перемещением
Чтобы данные о запасах оставались точными.

US-04
Как система WMS
Я хочу проверять наличие запасов перед подтверждением перемещения
Чтобы предотвратить ситуации с отрицательным балансом запасов.

US-05
Как система ERP
Я хочу получать подтверждение перемещения от WMS
Чтобы синхронизировать бухгалтерские остатки.

US-06
Как менеджер склада
Я хочу отслеживать остатки запасов в режиме реального времени по зонам склада
Чтобы я мог оптимизировать использование складских помещений.

US-07
Как менеджер склада
Я хочу видеть историю перемещений запасов с отметками времени и идентификаторами пользователей
Чтобы я мог проводить аудит складских операций.

US-08
Как оператор склада
Я хочу получать оповещения, если я сканирую не тот товар или место хранения
Чтобы ошибки предотвращались немедленно.

US-09
Как системный администратор
Я хочу иметь контроль доступа на основе ролей в WMS
Чтобы пользователи получали доступ только к функциям, соответствующим их роли.

US-10
Как менеджер склада
Я хочу видеть отклонения плана от фактического для перемещений запасов
Чтобы я мог выявлять узкие места в процессе.

US-11
Как оператор склада
Я хочу, чтобы система автоматически обновляла остатки запасов после подтверждения
Чтобы минимизировать ручные корректировки.

US-12
Как аналитик бизнес-аналитики
Я хочу, чтобы данные о перемещениях были доступны для отчетности в Power BI
Чтобы можно было анализировать операционные KPI




User story
US-01
As a Warehouse Planner
I want to create a stock movement request in ERP
So that warehouse operators receive a formal task.

US-02
As a Warehouse Operator
I want to see assigned movement tasks on my mobile device (TSD)
So that I can execute them without paper documents.

US-03
As a Warehouse Operator
I want to scan product barcodes before movement
So that stock data remains accurate.

US-04
As a WMS System
I want to validate stock availability before confirming movement
So that negative stock situations are prevented.

US-05
As an ERP System
I want to receive movement confirmation from WMS
So that accounting balances are synchronized.

US-06
As a Warehouse Manager
I want to monitor real-time stock balances by warehouse zone
So that I can optimize storage utilization.

US-07
As a Warehouse Manager
I want to see the history of stock movements with timestamps and user IDs
So that I can audit warehouse operations.

US-08
As a Warehouse Operator
I want to receive alerts if I scan the wrong product or location
So that errors are prevented immediately.

US-09
As a System Administrator
I want role-based access control in WMS
So that users only access functions relevant to their role.

US-10
As a Warehouse Manager
I want to see plan-fact deviations for stock movements
So that I can identify process bottlenecks.

US-11
As a Warehouse Operator
I want the system to auto-update stock balances after confirmation
So that manual corrections are minimized.

US-12
As a BI Analyst
I want movement data to be available for reporting in Power BI
So that operational KPIs can be analyzed.
