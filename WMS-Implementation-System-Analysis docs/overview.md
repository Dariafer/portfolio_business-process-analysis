Описание проекта
Данное исследование представляет собой анализ системы управления складом (WMS) для внедрения системы управления складом (WMS) в центре обработки металла. Бизнес-область сосредоточена на складской логистике и процессе внутреннего перемещения металлопродукции между зонами хранения.
Цель проекта — проанализировать текущий процесс перемещения на складе (AS IS), выявить узкие места и риски, а также разработать оптимизированную целевую модель (TO BE) с четко определенными системными требованиями, структурами данных и логикой интеграции между ERP и WMS.
Проект включает моделирование процессов, спецификацию требований, моделирование данных и проектирование интеграции API.

Бизнес-контекст
Складские операции включают частое перемещение металлопродукции между зонами хранения, зонами погрузки и производственными участками. Неточные данные о запасах, ошибки ручного ввода и задержки синхронизации между системами могут привести к расхождениям в инвентаризации, задержкам отгрузки и снижению прозрачности операций.

В рамках данного проекта предлагается структурированное системное решение, в котором:
Заказы на перемещение создаются в ERP-системе
Система управления складом (WMS) проверяет наличие товара на складе
Персонал склада подтверждает действия с помощью мобильных устройств (TSD)
Остатки запасов обновляются автоматически
Все операции регистрируются и отслеживаются

Project Description
This project represents a system analysis case for the implementation of a Warehouse Management System (WMS) in a metal service center. The business domain focuses on warehouse logistics and the process of internal movement of metal products between storage zones.
The objective of the project is to analyze the current warehouse movement process (AS IS), identify bottlenecks and risks, and design an optimized target model (TO BE) with clearly defined system requirements, data structures, and integration logic between ERP and WMS.
The case includes process modeling, requirements specification, data modeling, and API integration design.

Business Context
Warehouse operations involve frequent movement of metal products between storage zones, loading areas, and production sections. Inaccurate stock data, manual input errors, and delayed synchronization between systems can lead to inventory discrepancies, shipment delays, and reduced operational transparency.
This project proposes a structured system solution where:
Movement orders are created in ERP
WMS validates stock availability
Warehouse staff confirms actions via mobile devices (TSD)
Stock balances are updated automatically
All operations are logged and traceable
